# Story 13.6 — Greenfield Handler: New Project Bootstrap

## Story Statement

**As a** developer using Bob mode to start a brand new project,
**I want** Bob to orchestrate the complete project bootstrap sequence (4 phases),
**so that** I can go from empty directory to first story implementation without manual setup.

## Acceptance Criteria

### AC-1: Greenfield State Detection
- [x]`GreenfieldHandler` triggers only when `BobOrchestrator.detectProjectState()` returns `GREENFIELD`
- [x]Detection criteria: none of `package.json`, `.git`, `docs/`, `src/`, `.aios-core/` exist

### AC-2: Pre-Phase 0 GO/NO-GO
- [x]Surfaces BEFORE Phase 0 with welcome message + time estimate (no files created, no agents spawned)
- [x]Returns `{ action: 'greenfield_welcome', phase: 'detection', data: { message, options: ['GO', 'NO-GO'], timeEstimate, ... } }`
- [x]GO: proceeds to Phase 0 (Bootstrap) via `handleUserDecision(true, context)`
- [x]NO-GO: returns `{ success: false, reason: 'user_declined', action: 'greenfield_declined', data: {...} }`
- [x]`BobOrchestrator.handleGreenfieldDecision(accepted, context)` proxies to `greenfieldHandler.handleUserDecision()`

### AC-3: Phase 2 — Discovery
- [x]Executes requirements gathering phase
- [x]Surfaces to user between phases for GO/NO-GO confirmation

### AC-4: Phase 3 — Sharding
- [x]Executes PRD + architecture breakdown
- [x]Surfaces to user between phases for GO/NO-GO confirmation

### AC-5: Phase 4 — Dev Cycle
- [x]Executes first story implementation
- [x]Connects to standard `WorkflowExecutor` dev cycle
- [x]On completion returns `{ action: 'greenfield_dev_cycle', phase: 'phase_3_dev_cycle', data: { success: true, phasesCompleted: 4, nextStep: 'development_cycle', ... } }`
- [x]Semantic: Phase 3 is a handoff to dev cycle, not a terminal completion

### AC-6: Error Handling
- [x]Phase failure presents options: Retry / Skip / Abort (SurfaceChecker C004)
- [x]Abort saves session checkpoint — can resume from last completed phase

### AC-7: Idempotency / Resume
- [x]Re-run detects last completed phase from `SessionState`
- [x]Resumes from next uncompleted phase — does not restart from Phase 1
- [x]A phase that started but did not complete is treated as uncompleted — re-run retries it from the beginning (no partial-phase skip)

### AC-8: Observability Callbacks
- [x]Phase start/end events emitted to ObservabilityPanel
- [x]Progress percentage updated per phase (25% per phase)

### AC-9: Test Coverage
- [x]`tests/core/orchestration/greenfield-handler.test.js` covers state detection, all 4 phases, NO-GO at Pre-Phase 0 (pre-flight welcome), error handling, resume, and success return shape
- [x]Coverage >= 85% on `greenfield-handler.js`
