## Story

**As a** developer completing the SYNAPSE integration,

### domain-injector.js Design (@architect decision 2026-03-02)

**Mechanism:** File-write injection → `.aios/active-domains.md`

@architect (Aria) defined this mechanism in the architecture review session:
- Domain injector generates content as a formatted string
- Writes to `.aios/active-domains.md` — a session file
- `greeting-builder.js` reads this file as supplemental context during agent activation
- A PostCompact hook can re-run the injector when agent changes

**Output format for `.aios/active-domains.md`:**
```markdown
# SYNAPSE Active Domains
<!-- Generated by DomainInjector — do not edit manually -->
<!-- Updated: {ISO timestamp} -->

# constitution (L0)
{content of constitution domain file}

---

# global (L1)
{content of global domain file}

---

# agent-dev (L2)
{content of agent-dev domain file}

---

# commands (L7)
{content of commands domain file}
```

[Source: @architect gate decision, PRD-SYNAPSE-001 v0.2 Section 4]

### index.js Public API

This is the stable API that Epic 11 stories will consume:

```javascript
const {
  ManifestParser,           // parse .synapse/manifest, getActiveDomains()
  DomainInjector,           // inject domains, write session file
  SynapseMemoryProvider,    // bracket-based memory retrieval with cache
  createMemoryLoader,       // factory: new MemoryLoader(projectDir)
  createSelfLearner,        // factory: new SelfLearner(projectDir, options)
} = require('.aios-core/core/synapse');
```
