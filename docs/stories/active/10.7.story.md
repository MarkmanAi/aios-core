### domain-injector.js Design (@architect decision 2026-03-02)

**Mechanism:** File-write injection → `.aios/active-domains.md`

@architect (Aria) defined this mechanism in the architecture review session:
- Domain injector generates content as a formatted string
- Writes to `.aios/active-domains.md` — a session file
- `greeting-builder.js` reads this file as supplemental context during agent activation
- A PostCompact hook can re-run the injector when agent changes

**Output format for `.aios/active-domains.md`:**
```markdown
# SYNAPSE Active Domains
<!-- Generated by DomainInjector — do not edit manually -->
<!-- Updated: {ISO timestamp} -->

# constitution (L0)
{content of constitution domain file}

---

# global (L1)
{content of global domain file}

---

# agent-dev (L2)
{content of agent-dev domain file}

---

# commands (L7)
{content of commands domain file}
```

[Source: @architect gate decision, PRD-SYNAPSE-001 v0.2 Section 4]

This is the stable API that Epic 11 stories will consume:

const {

### *synapse status / *synapse-diagnose Wiring

**Decision (@po validation 2026-03-04):** Create a dedicated `synapse` agent at `.aios-core/development/agents/synapse.md`.

`*` commands in AIOS are defined in agent YAML `commands:` sections — not in `bin/aios.js` or `src/`. The `src/` directory does not exist in this project.

**@dev must:**
1. Create `.aios-core/development/agents/synapse.md` with at minimum:
   - `commands: [synapse-status, synapse-diagnose]`
   - Persona: SYNAPSE system agent
2. Implement the command handlers as documented in the agent file
3. The agent file follows the same YAML structure as other agents in `.aios-core/development/agents/`

**Note:** IDE sync (`ideSync` in core-config.yaml) will auto-propagate the new agent to `.claude/commands/AIOS/agents/`, `.cursor/rules/agents/`, and `.antigravity/rules/agents/` — @devops must run sync after agent creation.
