# Story 10.5 — SYNAPSE: Session Digest: PreCompact Hook Migration

## Status

Draft

## Story

**As a** developer integrating the MIS session digest system,
**I want** `session-digest/extractor.js` migrated and the PreCompact hook registered,
**so that** every context compact captures session knowledge before it is lost.

## Acceptance Criteria

1. `extractor.js` (386 lines) migrated to `.aios-core/core/synapse/memory/session-digest/extractor.js` with `require('yaml')` → `require('js-yaml')` and all yaml API calls updated
2. `tests/core/synapse/memory/session-digest/extractor.test.js` **created by @dev** (no source test file exists — write from scratch)

4. Hook returns in < 50ms (fire-and-forget via `setImmediate` — hook runner does not block)
5. Digest extraction completes in < 5 seconds
6. Output digest format matches YAML v1.0 schema: `{ schema_version, session_id, timestamp, corrections, patterns, axioms, context_snapshot }`
7. Graceful degradation: if feature gate disabled, hook logs "SYNAPSE digest skipped" and returns immediately — compact is never blocked
8. Integration test covers full flow: hook fires → extractor runs → digest file created in `.aios/session-digests/`
9. `npm test -- tests/core/synapse/memory/session-digest/` passes with zero failures
10. `npm run lint` passes on all new/modified files

### Fire-and-Forget Pattern (Critical for AC: 4)

The PreCompact hook must return to Claude Code in < 50ms. The actual digest extraction (< 5s) runs async.

Pattern using `setImmediate`:
```javascript
// Hook entry point — returns immediately to Claude Code
setImmediate(async () => {
  // This runs AFTER hook returns
  await extractSessionDigest(/* ... */);
});
// Hook returns here — well within 50ms
```

This is the same pattern used in Node.js for deferring heavy work without blocking the event loop.
[Source: PRD-SYNAPSE-001 v0.2 FR1, NFR1/NFR2]
