# Story 20.2 — MMOS Governance Adapter Pipeline

## Story Statement

**As a** Neo Strategic Council operator,
**I want** the MMOS pipeline to support a Governance Advisor mode (Tier B) that processes book-first sources with adapted L6-L7 inference and a framework-scoped system prompt,
**so that** authors like Meadows, Bungay, Skelton & Pais, and Grove can be invoked as governance advisors by Neo's `*war-room` command without violating P3/P6 principles.

---

## Context

**Origin:** Neo architectural decision 2026-03-11 — two-tier clone standard (`.neo/data/clone-standards.md`).

**The problem:** The current MMOS pipeline requires 3+ sources, APEX ≥ 70, and full L6-L8 personality data. This works for living people with rich source profiles (podcasts, interviews, social media). It fails for book-first or posthumous authors who are primarily accessible through written works.

**The solution:** A Governance Advisor mode that:
- Accepts 2+ written sources (books, papers, long-form articles)
- Uses APEX threshold of 55 (not 70)
- Extracts L1-L5 richly from text, L8 from ETL contradictions output
- Infers L6-L7 from narrative/autobiographical passages with explicit disclaimer
- Generates a system prompt scoped to frameworks & reasoning (not full persona)
- Labels the output as `[GOVERNANCE ADVISOR MODE]` so @emulator adapts its invocation

**This story depends on Story 20.1** — which fixes the ETL `--advisor` flag and produces proper `voice_dna.json`, `thinking_dna.json`, and `contradictions.json` as source inputs.

**Standard reference:** `.neo/data/clone-standards.md` — Tier B specification

**First target:** `donella-h-meadows` — sources already at `squads/mmos-squad/minds/donella-h-meadows/sources/books/thinking-in-systems/`

## Acceptance Criteria

### AC-1: `--tier` flag added to @mind-mapper

The mind-mapper agent (`squads/mmos-squad/agents/mind-mapper.md`) must document and support a `--tier governance-advisor` mode that modifies pipeline behavior.

- [ ] `mind-mapper.md` documents `--tier governance-advisor` as a valid invocation option
- [ ] When `--tier governance-advisor` is set:
  - APEX threshold: 55 (not 70)
  - Minimum sources: 2 written (not 3+ of any type)
  - L6-L7 pipeline step: inference from text (not standard collection)
  - L8 pipeline step: uses ETL `contradictions.json` as PRIMARY input if available
  - System prompt scope: frameworks & reasoning (not full persona)
- [ ] Standard `*map {name}` (no flag) runs existing Tier A pipeline unchanged
- [ ] `*map {name} --tier governance-advisor` triggers Tier B pipeline

### AC-2: Identity Analyst adapted for L6-L7 text inference

The `identity-analyst.md` agent (`squads/mmos-squad/agents/identity-analyst.md`) must handle a text-inference mode for L6-L7.

- [ ] `identity-analyst.md` documents a `governance-advisor` variant for L6-L7
- [ ] In this variant, the Identity Analyst:
  - Reads the book sources at `minds/{slug}/sources/books/`
  - Extracts L6 (values) from: prefaces, dedications, moral conclusions, autobiographical passages, author's stated principles
  - Extracts L7 (obsessions) from: recurring themes, topics the author returns to repeatedly, concerns expressed throughout the work
  - Marks ALL L6-L7 inferences with confidence level: `HIGH` (direct textual evidence) | `MEDIUM` (inferred from patterns) | `LOW` (speculative)
  - Does NOT fabricate biographical data not present in the sources
- [ ] Human checkpoint still occurs at L6-L8 — validator reviews framework accuracy, not persona fidelity
- [ ] Checkpoint prompt reads: "Validate that the extracted frameworks and reasoning patterns accurately represent the author's work. This is NOT a personality fidelity test."

### AC-3: System Prompt Architect generates Governance Advisor system prompt

The `system-prompt-architect.md` agent must generate a Tier B system prompt with mandatory disclaimer and framework scope.

- [ ] `system-prompt-architect.md` documents `governance-advisor` output mode
- [ ] Generated system prompt MUST begin with:
  ```
  [GOVERNANCE ADVISOR MODE]
  This is a framework-based advisor, not a full personality clone.
  Emulation scope: reasoning patterns, mental models, decision frameworks.
  L6-L7 (personal values, biography) inferred from written work only.
  Confidence: HIGH for frameworks | MEDIUM for values | LOW for conversational style.
  ```
- [ ] System prompt focuses on: reasoning patterns, mental models, decision heuristics, frameworks extracted from L1-L5 analysis
- [ ] System prompt does NOT attempt to emulate: conversational tone, personality quirks, how the person would respond in informal settings
- [ ] System prompt includes the relevant L2 KB frameworks from `.neo/kb/strategic/{domain}.md` in a `<governance_frameworks>` section
- [ ] Output file: `minds/{slug}/system_prompts/governance-advisor-v1.0.md`

### AC-4: @emulator auto-detects Governance Advisor mode

The `emulator.md` agent must detect the `[GOVERNANCE ADVISOR MODE]` marker and adapt its invocation accordingly.

- [ ] `emulator.md` documents Governance Advisor detection logic
- [ ] When @emulator loads a system prompt containing `[GOVERNANCE ADVISOR MODE]`:
  - Invocation preamble: "I will respond as [Name]'s reasoning framework and mental models would approach this problem:"
  - Invocation preamble (Tier A, no change): "I will respond as [Name] would:"
  - Response style: structured analysis using the author's frameworks — NOT simulated personal speech
  - @emulator does NOT attempt voice matching for Tier B advisors
- [ ] `*war-room` invocation via Neo always uses Governance Advisor mode (frameworks only)
- [ ] Standard `@emulator {name}` invocation respects whatever tier the loaded system prompt declares

### AC-5: Meadows Governance Advisor pipeline run end-to-end

- [ ] Sources available: `squads/mmos-squad/minds/donella-h-meadows/sources/books/thinking-in-systems/` (from Story 20.1)
- [ ] Run `*map donella-h-meadows --tier governance-advisor`
- [ ] All pipeline stages complete:
  - [ ] L1-L5: Cognitive Analyst produces 8-layer analysis from book sources
  - [ ] L6-L7: Identity Analyst infers from text with confidence levels
  - [ ] L8: Uses `contradictions.json` from ETL sources as primary input
  - [ ] Human checkpoint: framework validation (you validate)
  - [ ] System prompt generated: `governance-advisor-v1.0.md` with `[GOVERNANCE ADVISOR MODE]` header
- [ ] APEX score recorded (target: ≥ 55)
- [ ] ORGANOGRAMA entry updated by Neo: Meadows status `📚 PENDING pipeline` → `📚 GOVERNANCE ADVISOR`

### AC-6: No regression on existing Tier A pipeline

- [ ] `*map {any-existing-mind}` (no `--tier` flag) runs exactly as before — zero behavioral changes
- [ ] `*roundtable`, `*duo` commands work unchanged for Tier A minds
- [ ] Existing Tier A system prompts (marty_cagan, jeff_patton, kent_beck, etc.) unaffected

### L8 Input for Governance Advisors

If ETL `contradictions.json` exists at `minds/{slug}/sources/books/{book-slug}/contradictions.json`:
→ System Prompt Architect reads this file as PRIMARY L8 input
→ Cognitive Analyst / Identity Analyst supplement with their own L8 analysis
→ The ETL contradictions are high quality (verified: Meadows' contradictions.json has 2 rich paradoxes with verbatim quotes)

If no ETL contradictions exist:
→ Standard MMOS L8 extraction from sources (no change)

### KB Integration in System Prompt

The System Prompt Architect must inject the relevant L2 frameworks from `.neo/kb/strategic/` into the Governance Advisor system prompt. Example for Meadows:

```
<governance_frameworks source=".neo/kb/strategic/systems-thinking.md">
  [Content of systems-thinking.md injected here]
</governance_frameworks>
```

This ensures that when Neo invokes `*war-room` and @emulator loads Meadows, the systems thinking frameworks are immediately available in context — not requiring a separate KB lookup.

### Human Checkpoint Instruction (Tier B)

The standard L6-L8 human checkpoint message for Tier B reads:

```
[GOVERNANCE ADVISOR CHECKPOINT]
Reviewing: [Name] — Tier B Governance Advisor
Sources: [list of book sources]

Please validate:
1. Are the extracted frameworks accurate to the author's work?
2. Are the reasoning patterns correctly characterized?
3. Are the L6-L7 inferences (values/obsessions) plausible from the text?
   (Mark any that seem fabricated or overstated)
4. Does the Gold Layer (L8 contradictions) capture real tensions in the work?

This is NOT a personality fidelity test. We are validating frameworks, not persona.
Approve / Request changes:
```

### APEX Calculation for Tier B

The APEX score formula doesn't change — only the threshold:
- Tier A: APEX ≥ 70 required
- Tier B: APEX ≥ 55 accepted

Document in `mind-mapper.md` that Tier B APEX scores in the 55-70 range are acceptable and should be logged with a `[GOVERNANCE ADVISOR: APEX {score}]` notation.
