# Story 20.1 — ETL --advisor flag + L3 prompt fixes

## Story Statement

**As a** knowledge pipeline operator,
**I want** to replace the broken `--clone` flag with a working `--advisor` flag in the ETL CLI, with fixed L3 prompts that correctly populate all DNA fields,
**so that** book sources are properly prepared for the MMOS Governance Advisor pipeline without empty or incomplete extractions.

---

## Context

**Origin:** Neo architectural decision 2026-03-11 — two-tier clone standard (`.neo/data/clone-standards.md`).

**Problem diagnosed:**
The `--clone` flag in `knowledge-etl/src/knowledge_etl/cli.py` activated L3 extraction (Voice DNA + Thinking DNA + Contradictions). After processing "Thinking in Systems" (Meadows), two out of three extractions produced empty output:

- `voice_dna.json` — ALL structured fields empty (`signature_vocabulary: []`, `sentence_pattern: ""`, etc.). Only `chapter_observations` (free narrative) was populated.
- `thinking_dna.json` — COMPLETELY empty. All fields blank.
- `contradictions.json` — ✅ Rich content. 2 productive paradoxes with verbatim quotes.

**Root cause:** The XML prompts for voice and thinking DNA request conversational/personality data that doesn't exist in book text. The schemas have fields like "conversational style" and "tone markers" that require interview/podcast sources.

**Solution:** Redesign the prompts to extract only what text sources CAN provide: writing style patterns, structured reasoning, mental models, rhetorical devices. Rename `--clone` → `--advisor` to signal the new scope.

**Reference files:**
- `knowledge-etl/src/knowledge_etl/cli.py` — CLI entry point
- `knowledge-etl/src/knowledge_etl/transform/l3_authorial.py` — L3 logic
- `knowledge-etl/src/knowledge_etl/prompts/l3_voice_dna.xml` — broken prompt
- `knowledge-etl/src/knowledge_etl/prompts/l3_thinking_dna.xml` — broken prompt
- `knowledge-etl/src/knowledge_etl/prompts/l3_contradictions.xml` — working, DO NOT TOUCH
- `knowledge-etl/data/staging/thinking-in-systems/l3/` — current broken output for reference
- `.neo/data/clone-standards.md` — Tier B specification

### Prompt Engineering Principle
The key insight: book text provides WRITING patterns, not SPEAKING patterns.
- ✅ Extractable from text: vocabulary, sentence structure, rhetorical devices, reasoning patterns, mental models, analogies, contradictions
- ❌ NOT extractable from text: conversational tone, how they respond in real-time, personality quirks visible only in speech

New prompts must use quote-first extraction: extract 3-5 verbatim quotes first → then analyze the pattern. This is the same technique that made `contradictions.json` work.

### Rate Limit Note
The `--advisor` flag triggers L3 which uses Opus (200s sleep already in place in `l3_authorial.py`). No changes needed to rate limiting.
