# Story 13.11 — `aios bob explain [on|off]`: Educational Mode Toggle

## Story Statement

**As a** Bob mode user,
**I want** to toggle educational mode on or off from the CLI,
**so that** I can choose between concise output (default) and step-by-step explanations of Bob's reasoning.

## Acceptance Criteria

### AC-1: Enable Educational Mode
- [x] `aios bob explain on` sets `educational_mode: true` in user config via `setUserConfigValue('educational_mode', true)` (writes to `~/.aios/user-config.yaml` — persists across all projects and shell restarts)
- [x] Also sets `educational_mode: true` in session state via `setSessionOverride('educational_mode', true)` — only if an active session exists
- [x] Output: `"Educational mode: ON — Bob will now explain his reasoning step-by-step."`
- [x] Calling `explain on` when already ON is idempotent — same config write, same confirmation message

### AC-2: Disable Educational Mode
- [x] `aios bob explain off` sets `educational_mode: false` in user config via `setUserConfigValue('educational_mode', false)` (writes to `~/.aios/user-config.yaml`)
- [x] Also sets `educational_mode: false` in session state via `setSessionOverride('educational_mode', false)` — only if an active session exists
- [x] Output: `"Educational mode: OFF — Bob will be more concise from now on."`
- [x] Calling `explain off` when already OFF is idempotent — same config write, same confirmation message

### AC-3: Show Current State (No Arg)
- [x] `aios bob explain` (no argument) reads `educational_mode` from L5 user config via `getConfigAtLevel(process.cwd(), 'user')`
- [x] Prints: `"Educational mode: ON"` or `"Educational mode: OFF"`
- [x] Defaults to `OFF` if key is absent from user config (file missing or key not set)

### AC-4: Invalid Argument
- [x] Any argument other than `on`/`off` prints: `"Usage: aios bob explain [on|off]"` and exits with code 1
- [x] Exit code 0 for all valid usages (`on`, `off`, no arg)

### AC-5: Tests
- [x] `src/bob/__tests__/explain.test.js` covers: `explain on`, `explain off`, `explain` (no arg ON), `explain` (no arg OFF), invalid arg, active session sync, no session guard
- [x] `setUserConfigValue` and `getConfigAtLevel` are mocked
- [x] `SessionState` is mocked; `setSessionOverride` called only when `loadSessionState` returns non-null
- [x] Coverage >= 80% on `src/bob/commands/explain.js`

`setUserConfigValue` writes to `~/.aios/user-config.yaml` (L5 global user layer, cross-project). It is **synchronous** — no `await` needed.

```javascript

// Write (synchronous — no await):

// Read current state (AC-3):

```

**Note:** `getConfigAtLevel(root, 'user')` for L5 reads `~/.aios/user-config.yaml` using an absolute path — the `projectRoot` parameter is accepted but ignored for user-level config.

### CRITICAL: Session Override — Guard With `loadSessionState` First

`setSessionOverride()` throws if `this.state` is null. Always call `loadSessionState()` first and only set the override if a session exists:

```javascript
const { SessionState } = require('../../../.aios-core/core/orchestration/session-state');

const session = new SessionState(process.cwd());
const sessionData = await session.loadSessionState();
if (sessionData) {
  await session.setSessionOverride('educational_mode', enable);
}
// If no active session — skip silently (config write already persisted the choice)
```

**Note:** Session sync failure (e.g., `ENOENT`, schema errors) must be caught and swallowed — it is non-critical. The config write (`setUserConfigValue`) already persisted the setting.

[Source: `.aios-core/core/orchestration/session-state.js` lines 373–401]

### Full Implementation Pattern

```javascript
// src/bob/commands/explain.js
'use strict';

const { Command } = require('commander');
const chalk = require('chalk');
const { setUserConfigValue, getConfigAtLevel } = require('../../../.aios-core/core/config/config-resolver');
const { SessionState } = require('../../../.aios-core/core/orchestration/session-state');

async function runExplain(state) {
  const projectRoot = process.cwd();

  // AC-3: No argument — show current state
  if (state === undefined || state === null) {
    const userConfig = getConfigAtLevel(projectRoot, 'user') || {};
    const isOn = userConfig.educational_mode === true;
    console.log(`Educational mode: ${isOn ? chalk.green('ON') : chalk.yellow('OFF')}`);
    return;
  }

  // AC-5: Validate argument
  if (state !== 'on' && state !== 'off') {
    console.error('Usage: aios bob explain [on|off]');
    process.exit(1);
  }

  const enable = state === 'on';

  // AC-1/AC-2/AC-4: Write to user config (synchronous — persists across sessions)
  setUserConfigValue('educational_mode', enable);

  // AC-1/AC-2: Also sync to session state if a session is active
  try {
    const session = new SessionState(projectRoot);
    const sessionData = await session.loadSessionState();
    if (sessionData) {
      await session.setSessionOverride('educational_mode', enable);
    }
  } catch {
    // Session sync is non-critical — swallow errors
  }

  // AC-5: Confirmation message
  if (enable) {
    console.log(chalk.green('Educational mode: ON') + ' — Bob will now explain his reasoning step-by-step.');
  } else {
    console.log(chalk.yellow('Educational mode: OFF') + ' — Bob will be more concise from now on.');
  }
}

const explainCommand = new Command('explain')
  .description('Toggle educational mode on/off')
  .argument('[state]', 'on or off (omit to show current state)')
  .action(async (state) => {
    try {
      await runExplain(state);
    } catch (err) {
      console.error(chalk.red(`Bob explain error: ${err.message}`));
      process.exit(1);
    }
  });

module.exports = explainCommand;
module.exports.runExplain = runExplain;
```
