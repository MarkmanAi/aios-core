### 3-Layer Progressive Disclosure (memory-retriever.js)

The retriever implements auto-selection based on tokenBudget:

| Budget | Layer | Token Estimate | Description |
|--------|-------|----------------|-------------|
| < 500 | L1 | ~50 tokens | Metadata only |
| < 1500 | L2 | ~200 tokens | Relevant chunks |
| ≥ 1500 | L3 | ~1000+ tokens | Full content |

These thresholds are defined as constants in `TOKEN_ESTIMATES` and tested in the test file. Preserve exactly.
[Source: PRD-SYNAPSE-001 v0.2, FR2]

### Agent-Scoped Privacy

`memory-loader.js` enforces that each agent accesses only its own memories plus shared memories. The `dev` agent must NOT be able to retrieve memories scoped to `qa`. This is enforced via the `agentId` filter in the retriever.

Test for this: look for a test case like `"dev agent cannot access qa memories"` or similar in the test file.
[Source: PRD-SYNAPSE-001 v0.2, FR2]

### AGENT_SECTOR_PREFERENCES

`memory-loader.js` exports `AGENT_SECTOR_PREFERENCES` — a mapping of agent IDs to their preferred cognitive sectors:

```javascript
const AGENT_SECTOR_PREFERENCES = {
  dev: ['procedural', 'semantic'],
  // ... other agents
};
```

This constant must be preserved verbatim and is tested. Do not alter values.
