**Root causes documented:**
- **Exit 127 pattern (3 bash tests):** Tests call scripts via relative path (e.g., `bash assess-sources.sh`). When run from `scripts/tests/`, the scripts are not in PATH. Scripts ARE present in `scripts/` — this is a test runner path issue, not a script bug. Fix: run bash tests from `scripts/` directory, or update test scripts to use `../assess-sources.sh`.
- **ModuleNotFoundError (6 Python tests):** Tests import Python modules (`checklist_validator`, `dependency_check`, `inventory`, `naming_validator`, `security_scanner`, `yaml_validator`) that are not in `scripts/`. These are likely modules expected from a Python package not yet scaffolded. Tests were written ahead of implementation.
- **FileNotFoundError (2 Python tests):** `refresh-registry.py` and `squad-analytics.py` referenced as scripts but these are yaml tasks in `tasks/` directory — they don't have Python script equivalents. Tests may have been written expecting script implementation.
- **API mismatch (coherence_validator):** Test expects `overall_status` key in result but script returns `status`. Minor naming inconsistency.
- **Windows newline (quality_gate):** `count_lines("line1\nline2\nline3\n")` returns 3 on Windows instead of expected 4. Windows newline handling difference.
