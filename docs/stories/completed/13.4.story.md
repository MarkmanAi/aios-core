# Story 13.4 — MessageFormatter: Educational Mode

## Story Statement

**As a** Bob mode user who wants to learn how AIOS works,
**I want** to enable an educational mode that shows Bob's reasoning and agent delegations,
**so that** I can understand what is happening under the hood without needing advanced mode.

## Acceptance Criteria

### AC-1: Concise Mode (OFF — default)
- [ ] When `educationalMode: false`, messages contain only results/outcomes
- [ ] No agent names, reasoning, or trade-offs are included in output
- [ ] Output is brief and action-oriented

### AC-2: Educational Mode (ON)
- [ ] When `educationalMode: true`, messages include: which agent was delegated to, reasoning, trade-offs considered
- [ ] Output clearly labels the "why" behind each action
- [ ] Agent name is always shown in educational output

### AC-3: Mode Setting
- [ ] `setEducationalMode(true/false)` toggles the mode at runtime
- [ ] `getEducationalMode()` returns current state as boolean
- [ ] Mode can be toggled mid-session without restart

### AC-4: Mode API Contract
- [ ] `setEducationalMode(true/false)` updates internal boolean state immediately
- [ ] `getEducationalMode()` returns current boolean — usable by any consumer
- [ ] Persistence of educational mode is NOT MessageFormatter responsibility — owned by BobOrchestrator via `_resolveEducationalMode()` (Story 13.1 AC-6)

### AC-5: Output Contract
- [ ] `format(message, context)` always returns a `string` — never `null`, `undefined`, or throws for valid inputs
- [ ] `format(message, null)` and `format(message, undefined)` return a safe string (no crash) — context is optional
- [ ] In concise mode, return value contains only `message` — no agent, reasoning, or trade-off fields appended

### AC-6: Test Coverage
- [ ] `tests/core/orchestration/message-formatter.test.js` covers both modes
- [ ] Verifies concise output does NOT include agent/reasoning fields
- [ ] Verifies educational output DOES include agent/reasoning fields
- [ ] Verifies `format()` with null/undefined context does not throw
- [ ] Coverage >= 90% on `message-formatter.js`
