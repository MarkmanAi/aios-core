## Story

**so that** parallel Jest workers never collide on shared filesystem paths and `npm test` produces deterministic results on every run.

## Dev Notes

### Root Cause (Class A — Fixed Shared Paths)

All 5 suites write to hardcoded, non-unique paths. When Jest runs 2+ workers concurrently, both workers execute `rm -rf path` then `mkdir path` in `beforeAll`. The workers race: Worker A's `mkdir` succeeds, Worker B's `rm -rf` deletes it, Worker A tries to write → `ENOENT`.

**Mechanism:**
```
Worker A: beforeAll → rm('/fixed-path') → mkdir('/fixed-path')
Worker B: beforeAll → rm('/fixed-path') ← deletes what Worker A just created!
Worker A: test runs → ENOENT: no such file or directory '/fixed-path'
```

### Fix Pattern (Apply to ALL 5 files)

```javascript
// BEFORE (broken — shared path):
const TEST_DIR = path.join(os.tmpdir(), 'aios-terminal-spawner-test'); // FIXED name

// AFTER (correct — unique per invocation):
let tmpDir;
beforeAll(() => {
  tmpDir = fs.mkdtempSync(path.join(os.tmpdir(), 'aios-terminal-spawner-'));
});
afterAll(() => {
  fs.rmSync(tmpDir, { recursive: true, force: true });
});
```
