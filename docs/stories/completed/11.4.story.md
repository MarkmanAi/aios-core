## Dev Notes

### Root Cause (Class B — Mock State Leakage)

`jest.clearAllMocks()` resets:
- Mock call history (`.mock.calls`, `.mock.instances`, `.mock.results`)

`jest.clearAllMocks()` does NOT reset:
- `mockImplementation()` — custom implementations persist
- `mockReturnValue()` / `mockResolvedValue()` — return values persist
- `mockReturnValueOnce()` — queued return values accumulate

When a test sets `mockFn.mockImplementation(() => 'value-A')` and the next test expects the mock to return its default (undefined or jest.fn()), it gets 'value-A' instead — causing assertion failures that depend on which test ran first.

### File Analysis

**`tests/core/unified-activation-pipeline.test.js`:**
- 14+ mocks (including `fs`, `path`, `os`, agent module mocks)
- `fs.readFile` mock checks string paths — implementation persists across tests
- `globalConfigCache` — module-level cache object not reset between tests
- Fix: Replace `jest.clearAllMocks()` with `jest.resetAllMocks()` in `beforeEach`
- Additional: Find and reset `globalConfigCache` in `beforeEach`

**`tests/core/context-aware-greetings.test.js`:**
- 7 mocks (`ContextDetector`, `GitConfigDetector`, and others)
- Custom implementations for `ContextDetector` and `GitConfigDetector` not re-initialized per test
- Fix: Replace `jest.clearAllMocks()` with `jest.resetAllMocks()` in `beforeEach`
- Additional: Move all `mockImplementation()` calls into `beforeEach` for explicit re-initialization

### Fix Pattern

**Option A (Preferred — simpler):**
```javascript
// BEFORE:
beforeEach(() => {
  jest.clearAllMocks();
});

// AFTER:
beforeEach(() => {
  jest.resetAllMocks(); // resets implementations AND call history
});
```

**Option B (If resetAllMocks breaks existing tests):**
```javascript
beforeEach(() => {
  jest.clearAllMocks();
  // Re-apply all custom implementations explicitly:
  mockContextDetector.detect.mockImplementation(() => ({ type: 'new' }));
  mockGitConfigDetector.detect.mockImplementation(() => ({ name: 'test' }));
  // ... etc for all 7 mocks
});
```

**globalConfigCache reset (if applicable):**
```javascript
// Find where globalConfigCache is defined or imported in the test
// Add to beforeEach:
beforeEach(() => {
  jest.resetAllMocks();
  // If globalConfigCache is accessible:
  globalConfigCache.clear(); // or globalConfigCache = {};
  // If it's a module-level variable, use jest.isolateModules() or jest.resetModules()
});
```

### Caution: resetAllMocks Side Effects

Before applying `resetAllMocks()`, verify:
1. No test relies on a `mockReturnValueOnce()` queue built up across multiple `beforeEach` calls
2. No test relies on mock call counts from the `beforeAll` block (those would also be reset)
3. Check if any tests use `.toHaveBeenCalledTimes()` where the count spans multiple operations in `beforeAll` + the test itself

If any of these patterns exist, use Option B (explicit re-initialization) instead of Option A.

### Technical Constraints

- **NEVER** modify production code — all changes are in test files only
- If `jest.resetAllMocks()` causes previously-passing tests to fail, this reveals hidden mock state dependencies — document them in Dev Notes and use Option B
- Do not add `jest.resetModules()` without careful analysis — this re-requires all modules and may cause timing issues
