## Failures by Category

### Category A: Test Runner Path Issue (3 bash scripts)

**Scripts:** `test_assess_sources.sh`, `test_fidelity_score.sh` (partial), `test_validate_clone.sh`

**Symptom:** Exit code 127 — command not found when test tries to call the target script.

**Root cause:** Tests call scripts with relative path (e.g., `bash assess-sources.sh`) but are executed from `scripts/tests/` directory. The scripts exist in `scripts/` but not in `scripts/tests/`.

**Evidence:**
```
FAIL: gold_exit (expected exit 0, got 127)
```

**Fix options:**
1. Update test scripts to use `../assess-sources.sh` (relative path fix)
2. Run tests from `scripts/` directory (`bash tests/test_assess_sources.sh`)
3. Add `cd ..` at test script start

**Effort:** Low (1-2 lines per test script)

### Category B: Missing Python Modules (6 Python scripts)

**Scripts:** `test_checklist_validator.py`, `test_dependency_check.py`, `test_inventory.py`, `test_naming_validator.py`, `test_security_scanner.py`, `test_yaml_validator.py`

**Symptom:** `ModuleNotFoundError` at collection time.

**Root cause:** Tests import Python modules that don't exist:
- `checklist_validator` — not in `scripts/`
- `dependency_check` — not in `scripts/`
- `inventory` — not in `scripts/`
- `naming_validator` — not in `scripts/`
- `security_scanner` — not in `scripts/`
- `yaml_validator` — not in `scripts/`

**Assessment:** Tests were likely written ahead of implementation, or the modules were expected from a Python package not installed. The logic these tests cover may be implemented differently (e.g., in bash, or via external tools).

**Fix options:**
1. Implement missing Python modules (if the functionality is still needed)
2. Delete test files if functionality is covered differently
3. Mark tests as skip with `@pytest.mark.skip(reason="module not yet implemented")`

**Effort:** Medium to High (requires decision on whether to implement or remove)
