# Story 11.5 — CI Health: Test Isolation — Windows Timing, Env Mutation & Real File

## Status

Ready for Review

## Story

**As a** developer running `npm test` on Windows CI,
**I want** all timing races, environment variable leaks, and real-file dependencies eliminated from the test suite,
**so that** `npm test` produces deterministic results on Windows CI with zero EBUSY errors and zero env-state bleed-through between tests.

## Dev Notes

### Class C — Windows Timing Races

**`tests/installer/brownfield-upgrader.test.js`**

Current broken pattern:
```javascript
const tmpDir = path.join(os.tmpdir(), `aios-test-${Date.now()}`);
```
Risk: Two parallel workers starting within the same millisecond produce identical paths.

Fix:
```javascript
let tmpDir;
beforeAll(() => {
  tmpDir = fs.mkdtempSync(path.join(os.tmpdir(), 'aios-brownfield-upgrader-'));
});
afterAll(() => {
  fs.rmSync(tmpDir, { recursive: true, force: true });
});
```

---

**`tests/unit/documentation-integrity/brownfield-analyzer.test.js`**

Current broken pattern: `fs.rmSync()` called without retry — Windows EBUSY when a file handle is still open from a previous write.

Fix (wrap all rmSync calls):
```javascript
function rmSyncWithRetry(targetPath, options = {}, maxRetries = 3, delayMs = 100) {
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      fs.rmSync(targetPath, { recursive: true, force: true, ...options });
      return;
    } catch (err) {
      if ((err.code === 'EBUSY' || err.code === 'EPERM') && attempt < maxRetries - 1) {
        // Synchronous delay for test cleanup — acceptable in afterEach
        Atomics.wait(new Int32Array(new SharedArrayBuffer(4)), 0, 0, delayMs * (attempt + 1));
      } else {
        throw err;
      }
    }
  }
}
```

Or simpler (if Node ≥ 14.14): use `fs.rmSync` with `maxRetries` option:
```javascript
fs.rmSync(tmpDir, { recursive: true, force: true, maxRetries: 3, retryDelay: 100 });
```
Check Node version in `.nvmrc` or `package.json engines` before using `maxRetries` option.

---

**`packages/installer/tests/integration/environment-configuration.test.js`**

Current state: Already has retry logic with 50ms delay. Verify the delay is sufficient on slow CI.

Fix (if insufficient): Increase delay to 200ms or add a retry loop around the cleanup assertion.

### Class D — Real File Dependency

**`tests/core/surface-checker.test.js`**

Current broken pattern: Reads `.aios-core/core/orchestration/surface-criteria.yaml` from the real repo without mocking. If the file changes, tests may silently pass or fail based on real content.

Fix Option A (mock the file read):
```javascript
jest.mock('fs', () => ({
  ...jest.requireActual('fs'),
  readFileSync: jest.fn((filePath, encoding) => {
    if (filePath.includes('surface-criteria.yaml')) {
      return `# fixture content\ncriteria:\n  - name: test-criterion\n    type: required\n`;
    }
    return jest.requireActual('fs').readFileSync(filePath, encoding);
  }),
}));
```

Fix Option B (use a fixture file):
- Create `tests/core/fixtures/surface-criteria.yaml` with minimal valid content
- Mock the path resolution to point to the fixture instead of the real file

Option B is preferred — it keeps the mock simple and the fixture is version-controlled.

### Class E — process.env Mutation

**`tests/core/orchestration/terminal-spawner.test.js`** (env part — path part handled in Story 11.3)

Current broken pattern:
```javascript
process.env.GITHUB_ACTIONS = 'true';
process.env.VSCODE_GIT_IPC_HANDLE = '/tmp/vscode.sock';
// No restoration after test
```

Fix:
```javascript
let envSnapshot;
beforeEach(() => {
  envSnapshot = { ...process.env }; // shallow copy sufficient for string values
});
afterEach(() => {
  // Restore: delete keys added, restore changed keys
  Object.keys(process.env).forEach(key => {
    if (!(key in envSnapshot)) {
      delete process.env[key];
    }
  });
  Object.assign(process.env, envSnapshot);
});
```

Note: If Story 11.3 already modified this file for the path fix, apply the env fix in the same file — coordinate to avoid conflicts.

### Technical Constraints

- **NEVER** modify production code — all changes are in test files only
- Check Node.js version before using `fs.rmSync` `maxRetries` option (requires Node ≥ 14.14)
- `Atomics.wait` for synchronous delay requires `SharedArrayBuffer` — verify it's available in Jest's test environment. Alternative: use a simple busy-wait loop if not available.
- Story 11.3 already touches `terminal-spawner.test.js` — coordinate to avoid merge conflicts. If 11.3 is merged first, apply env fix as a follow-up commit to the same file.
