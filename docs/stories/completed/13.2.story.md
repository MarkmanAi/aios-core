# Story 13.2 — Data Lifecycle Manager

## Story Statement

**As a** AIOS operator running Bob over time,
**I want** Bob to automatically archive old sessions and clean up stale data on each run,
**so that** my workspace stays uncluttered and I never lose session history due to orphaned locks or disk bloat.

## Acceptance Criteria

### AC-1: Session State Archival
- [ ] Sessions older than 30 days are archived to `.aios/archive/sessions/`
- [ ] Archive directory is created automatically if it does not exist
- [ ] Archived sessions retain their original JSON structure
- [ ] Original session file is removed only after the archive write is confirmed successful (no data loss on partial failure — if deletion fails, original is preserved and error is logged)

### AC-2: Snapshot Cleanup
- [ ] Snapshots older than 90 days are removed from disk
- [ ] A reference entry is kept in `.aios/archive/snapshot-index.json`
- [ ] Index entry includes: `id`, `removed_at`, `original_path`
- [ ] Index file write is atomic — existing index entries are not lost on write failure (read → merge → write pattern)

### AC-3: Orphan Lock Cleanup
- [ ] Orphan lock files (process no longer running) are detected on startup
- [ ] Cleanup delegated to `LockManager.cleanupStaleLocks()`
- [ ] Count of removed locks is returned in lifecycle report

### AC-4: Lifecycle Report
- [ ] `runLifecycle()` returns `{ archivedSessions, removedSnapshots, cleanedLocks }`
- [ ] All counts are integers >= 0
- [ ] Error in one cleanup phase does not abort other phases

### AC-5: Edge Cases
- [ ] `runLifecycle()` with no sessions to archive returns `{ archivedSessions: 0, removedSnapshots: 0, cleanedLocks: 0 }` — no error thrown
- [ ] Archival of a session that fails during deletion preserves the original file and increments no counter (counts only fully-completed operations)

### AC-6: Test Coverage
- [ ] `tests/core/orchestration/data-lifecycle-manager.test.js` covers all 3 cleanup operations
- [ ] Error isolation test: one phase throwing does not block others
- [ ] Partial failure test: archive written but deletion fails — original file preserved
- [ ] Coverage >= 85% on `data-lifecycle-manager.js`
