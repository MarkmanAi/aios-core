# Story 13.12 — `aios bob switch`: Switch to Advanced Mode

## Story Statement

**As a** Bob mode user who has outgrown simplified mode,
**I want** to switch to advanced mode via a single command with a clear explanation,
**so that** I can access all AIOS agents and commands when I'm ready.

## Acceptance Criteria

### AC-1: Explanation Before Switch
- [x] `aios bob switch` prints what advanced mode means BEFORE asking for confirmation
- [x] Explanation includes: "Access to all 11 agents (@dev, @qa, @architect, etc.)" and "Full command palette"
- [x] Explanation is brief (max 6 lines)

### AC-2: Confirmation Prompt
- [x] Shows confirmation prompt: `"Switch to advanced mode? [y/N]"` (default: No)
- [x] Requires explicit `y` or `yes` (case-insensitive) to proceed
- [x] Any other input cancels: `"No changes made. Bob is still your interface."`

### AC-3: `--yes` Flag (Skip Confirmation)
- [x] `aios bob switch --yes` skips confirmation prompt and switches directly
- [x] Still prints the explanation (AC-1) before switching

### AC-4: Profile Toggle
- [x] On confirmation: calls `toggleUserProfile()` from `config-resolver.js` (no argument — toggles bob→advanced)
- [x] Written to `~/.aios/user-config.yaml` (L5 layer) via `setUserConfigValue('user_profile', 'advanced')` internally
- [x] Confirmation message: `"Done. You now have access to all agents. Use @dev, @qa, @architect, etc."`

### AC-5: Cancel Flow
- [x] On cancel: `"No changes made. Bob is still your interface."`
- [x] Exit code 0 (not an error)
- [x] Config is NOT modified (`toggleUserProfile` NOT called)

### AC-6: Tests
- [x] `src/bob/__tests__/switch.test.js` covers: explanation display, confirmation flow, `--yes` flag, cancel flow, `toggleUserProfile` call
- [x] `toggleUserProfile` is mocked to verify it IS called on confirm and NOT called on cancel
- [x] `@clack/prompts` `confirm` is mocked for confirmation flow tests
- [x] Coverage >= 80% on `src/bob/commands/switch.js`

### API Discrepancy: `toggleUserProfile()` Signature

> **CRITICAL**: The Epic 13 spec says `toggleUserProfile('advanced')` but the actual implementation does NOT accept arguments.

[Source: `.aios-core/core/config/config-resolver.js` lines 569–585]

```javascript
function toggleUserProfile() {
  // Reads current config, flips bob↔advanced, writes back
  const previous = config.user_profile || 'advanced';
  const current = previous === 'bob' ? 'advanced' : 'bob';
  setUserConfigValue('user_profile', current);
  return { previous, current };
}
```

**Implication for implementation**: Call `toggleUserProfile()` with NO argument. Since this story only switches bob→advanced (user is in bob mode), and the function toggles deterministically, it will always produce `advanced` when current profile is `bob`. No argument needed.

### Confirmation Prompt Library

[Source: docs/framework/tech-stack.md — @clack/prompts]

`@clack/prompts ^0.11.0` is confirmed in `package.json`. Use `confirm` from `@clack/prompts`:

```javascript
const { confirm } = require('@clack/prompts');

const shouldSwitch = await confirm({ message: 'Switch to advanced mode?' });
// Returns: true | false | symbol (if cancelled via Ctrl+C)
```

**Important**: `confirm` from clack returns `symbol` (not boolean) when the user cancels via Ctrl+C. Guard against this:

```javascript
if (shouldSwitch !== true) {
  // false OR symbol (Ctrl+C) → treat as cancel
  console.log('No changes made. Bob is still your interface.');
  return;
}
```

For `--yes` flag: skip `confirm` entirely, proceed directly to `toggleUserProfile()`.

### Implementation Pattern (from explain.js)

[Source: `src/bob/commands/explain.js`]

```javascript
'use strict';

const { Command } = require('commander');
const chalk = require('chalk');
const { toggleUserProfile } = require('../../../.aios-core/core/config/config-resolver');
const { confirm } = require('@clack/prompts');

const ADVANCED_MODE_EXPLANATION = `
Advanced mode gives you access to all 11 AIOS agents and the full command palette.

  Agents: @dev, @qa, @architect, @pm, @po, @sm + 5 more specialists.
  Full command palette — every CLI command becomes available.

You can still use Bob at any time: aios bob do "<request>"
`;

async function runSwitch(options = {}) {
  // AC-1: Always print explanation first
  console.log(ADVANCED_MODE_EXPLANATION);

  // AC-3: --yes skips confirmation
  if (!options.yes) {
    // AC-2: Confirmation prompt
    const shouldSwitch = await confirm({ message: 'Switch to advanced mode?' });
    if (shouldSwitch !== true) {
      // AC-5: Cancel flow
      console.log('No changes made. Bob is still your interface.');
      return;
    }
  }

  // AC-4: Toggle profile bob→advanced (no argument needed)
  toggleUserProfile();
  console.log(chalk.green('Done.') + ' You now have access to all agents. Use @dev, @qa, @architect, etc.');
}

const switchCommand = new Command('switch')
  .description('Switch from Bob mode to advanced agent mode')
  .option('--yes', 'Skip confirmation prompt')
  .action(async (options) => {
    try {
      await runSwitch(options);
    } catch (err) {
      console.error(chalk.red(`Bob switch error: ${err.message}`));
      process.exit(1);
    }
  });

module.exports = switchCommand;
module.exports.runSwitch = runSwitch;
```
