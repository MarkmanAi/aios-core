# Story 13.5 — Brownfield Handler: First-Run Discovery

## Story Statement

**As a** developer using Bob mode on an existing project without AIOS docs,
**I want** Bob to automatically discover my codebase and generate architecture + debt documentation,
**so that** I can start using AIOS without manually writing documentation first.

## Acceptance Criteria

### AC-1: State Detection
- [ ] `BrownfieldHandler` only triggers when `BobOrchestrator.detectProjectState()` returns `EXISTING_NO_DOCS`
- [ ] Does NOT trigger for `EXISTING_WITH_DOCS`, `GREENFIELD`, or `NO_CONFIG` states

### AC-2: Welcome Conversation
- [ ] Displays welcome message before execution begins
- [ ] Message includes time estimate: "4–8 hours"
- [ ] Uses `SurfaceChecker` (C002: GO/NO-GO) to confirm before starting
- [ ] If user answers NO-GO, `BrownfieldHandler` returns `{ success: false, reason: 'user_declined', action: 'brownfield_declined' }` — no workflow is executed

### AC-3: Workflow Execution
- [ ] Executes `brownfield-discovery.yaml` workflow via `WorkflowExecutor`
- [ ] Workflow runs all discovery phases to completion
- [ ] If `WorkflowExecutor` fails mid-workflow, returns `{ success: false, reason: 'workflow_error', action: 'brownfield_error', error: <message> }` — no output files are left in partial state (partial files are removed)

### AC-4: Outputs
- [ ] Generates `system-architecture.md` at `docs/architecture/system-architecture.md`
- [ ] Generates `TECHNICAL-DEBT-REPORT.md` at `docs/reports/TECHNICAL-DEBT-REPORT.md`
- [ ] Both files are valid markdown with at minimum: one `#` heading and one non-empty content section

### AC-5: Post-Discovery Flow
- [ ] After completion, presents user with options: resolve debts OR add new feature
- [ ] Uses `SurfaceChecker` (C003) to present the options

### AC-6: Idempotency
- [ ] Re-running on a project that already has docs does NOT re-execute discovery
- [ ] Existing `system-architecture.md` is overwritten (full replacement, not appended) on re-run
- [ ] Returns `{ success: true, action: 'updated' }` on idempotent re-run (detection via `checkIdempotency()` before calling `_executeDiscovery()`)

### AC-7: Test Coverage
- [ ] `tests/core/orchestration/brownfield-handler.test.js` covers state detection, workflow execution, outputs, idempotency, and NO-GO path
- [ ] Coverage >= 85% on `brownfield-handler.js`
