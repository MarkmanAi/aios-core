### no-useless-escape Detail
`\Z` is not a recognized escape in JS regex. It's treated as just `Z`. Safe to remove the backslash. Example:
```js
// Before (warning)
content.match(/output: \|\n([\s\S]+\Z)$/)
// After (clean)
content.match(/output: \|\n([\s\S]+Z)$/)
```
Verify context — if `\Z` was intended as end-of-string (like Python regex), the fix may need to be `$` instead.

### no-case-declarations Detail
```js
// Before (warning)
case 'foo':
  const x = 1;  // ← warning: no-case-declarations
  break;

// After (clean)
case 'foo': {
  const x = 1;
  break;
}
```
