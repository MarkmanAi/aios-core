# Story 13.1 — Bob Orchestrator: Decision Tree Entry Point

## Story Statement

**As a** developer running AIOS for the first time on any project,
**I want** the BobOrchestrator to detect my project state and route me to the correct setup or dev flow automatically,
**so that** Bob takes the right action without requiring me to know which mode to invoke manually.

## Acceptance Criteria

### AC-1: Project State Detection
- [ ] `detectProjectState()` returns one of four states: `NO_CONFIG`, `EXISTING_NO_DOCS`, `EXISTING_WITH_DOCS`, `GREENFIELD`
- [ ] Detection based on presence/absence of `.aios-core/`, `docs/`, `package.json`, `.git`
- [ ] Returns `NO_CONFIG` when `.aios-core/` is not found in project root

### AC-2: Routing Logic
- [ ] `orchestrate(userInput)` routes to `GreenfieldHandler` when state is `GREENFIELD`
- [ ] Routes to `BrownfieldHandler` when state is `EXISTING_NO_DOCS`
- [ ] Routes to standard `WorkflowExecutor` dev cycle when state is `EXISTING_WITH_DOCS`
- [ ] Returns `{ success: false, error: 'aios-core not installed', state: 'NO_CONFIG' }` when state is `NO_CONFIG` — no further routing attempted
- [ ] Returns structured `OrchestrationResult` for all paths

### AC-3: Concurrency Protection
- [ ] A second `orchestrate()` call while one is running returns `{ success: false, error: 'already running' }`
- [ ] Lock is acquired at start and released on completion (success or error)
- [ ] Stale locks (process dead) do not block new executions

### AC-4: Observability Integration
- [ ] Phase changes are emitted to `ObservabilityPanel` via registered callbacks
- [ ] Agent assignments are emitted to `BobStatusWriter` on each routing decision
- [ ] Dashboard emitter fires events matching `BobEventTypes` enum

### AC-5: Test Coverage
- [ ] `tests/core/orchestration/bob-orchestrator.test.js` covers all 4 routing branches (including `NO_CONFIG` early return)
- [ ] Concurrency test verifies lock blocks second call
- [ ] Coverage >= 85% on `bob-orchestrator.js`

### AC-6: Educational Mode Resolution
- [ ] `_resolveEducationalMode()` reads `educational_mode` key from `SessionState` on startup
- [ ] Returns `false` as default when no value exists in session
- [ ] Passes resolved value to `MessageFormatter.setEducationalMode()` during initialization

### OrchestrationResult Schema
Every return from `orchestrate()` must conform to this shape:
```javascript
{
  success: boolean,          // true on successful routing completion
  state: ProjectState,       // detected project state
  handler: string | null,    // 'GreenfieldHandler' | 'BrownfieldHandler' | 'WorkflowExecutor' | null
  error: string | null,      // present only when success: false
  result: object | null,     // downstream handler result, or null
}
```
